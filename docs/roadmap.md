# Roadmap

## Publication Roadmap

The publication goal is not just “make the repo public.” The goal is to make it legible, credible, and durable as a portfolio project.

Near-term priorities:
- maintain a clean public boundary
- keep docs and generated evidence aligned
- expand the curated figure/report pipeline
- make the GitHub Pages surface stronger over time

## Research Roadmap

### 1. Repair data completeness where it matters most

Highest-value gap:
- open-interest and liquidation-family coverage for the target windows

Reason:
- these features should matter for regime and event discrimination
- current null / degraded coverage weakens confidence in downstream conclusions

### 2. Re-run matched timeframe comparisons

Priority comparison:
- 1h vs 4h under the same reporting and governance standard

Reason:
- the repo currently contains evidence from both styles of evaluation
- the public story becomes stronger when the trade-off is presented directly

### 3. Continue bounded threshold and governance calibration

Priority:
- tune for trade density without letting PF / Sharpe collapse
- keep hard risk gates strict
- only recalibrate soft enforcement where the taxonomy supports it

### 4. Clarify the Chronos track

Priority questions:
- does real-backend Chronos add value beyond the baseline stack?
- under what data completeness and regime conditions does it help or fail?

The project should be explicit here: the repo is named for Chronos-PLG, but the strongest currently inspected candidate evidence is still EWMA-led.

## Portfolio Roadmap

Useful post-publication enhancements:
- add a richer static report or lightweight interactive explorer
- add a secondary asset such as ETH once BTC framing is stable
- add deeper comparative experiment cards
- add publication automation around docs and figure refresh

## Non-Roadmap Items

Things intentionally not prioritized right now:
- pretending live deployment is near
- adding unnecessary infrastructure
- optimizing for flashy dashboards before the evidence layer is coherent
